**Routing rule (applies to every trigger below):** when the user types a skill's `/command` OR matches one of its listed trigger phrases, invoke the Skill tool with that `skill:` before doing anything else. This one rule replaces the per-skill "invoke the Skill tool" line each block used to repeat. The `# skill-discovery` block below is the sole exception — it is a routing rule, not a skill, so do NOT invoke the Skill tool for it.

- **eli5** — `/eli5`: explain anything in plain English before you commit to it. (routes via SKILL.md description)
- **my-skills** — `/my-skills`: list skills you authored (not plugin packs) + relationship map. (routes via SKILL.md description)
- **improve** — `/improve`: read-only senior-advisor audit across 9 categories (correctness, security, perf, tests, tech debt, deps, DX, docs, direction) → vetted findings → self-contained implementation plans for an executor model. (routes via SKILL.md description)
- **lockstep** — `/lockstep`, "lock step", "hold off on code", "don't code yet": hook-enforced gate: blocks Edit/Write/NotebookEdit until you say go. Not just a prompt reminder — mechanically denied.
- **plan** — `/plan`, "plan this", "help me think through", "plan before we build", "interview me about": pre-build planning interview: extracts design decisions, edge cases, constraints one question at a time before any code.
- **build** — `/build`, "build", "build this feature end to end": full feature pipeline: evidence → plan → Opus plan → approval gate → UI mockup gate → TDD → build → regression gate → prove.
- **tdd** — `/tdd`, "write tests first", "TDD this", "test-driven", "implement with tests": vertical-slice TDD — one test → one implementation → one green bar → repeat. Forbids all-tests-first horizontal slicing.
- **drawio-skill** — `/drawio-skill`, or asks for a diagram/flowchart/architecture diagram/ER diagram/UML diagram/sequence diagram: generates diagrams (flowchart/architecture/ER/UML/sequence/network/mind-map) as .drawio XML, exports PNG/SVG/PDF/JPG via the native draw.io desktop CLI.
- **my-md** — `/my-md`: list all markdown files: global ~/.claude/ docs + current project artifacts. (routes via SKILL.md description)
- **whats-next** — `/whats-next`, "what's next", "what should I do", "now what": checks task queue first, runs next task; if empty, scans project and proposes. Never shows a static menu.
- **debate** — `/debate`, "debate this", "stress test this decision", "get the team on this", "should we build this": 7-persona product-team debate: Senior Dev, Junior Dev, Sales Engineer, DevOps, Sales Leader, Eng Leader, Product Designer → Chairman oversight (evidence rulings, forced cross-examinations) → contradiction map → synthesis → peer review. Works on architecture, UI/UX, or feature-priority calls. Flag `--ui` swaps in a 5-persona UI/UX panel (Restraint Auditor, Operator, Spatial Designer, Accessibility Enforcer, Diagnostician) grounded in the project's design docs.
- **update** — `/update`, "check for updates", "am I out of date", "what's new", "update my setup", "rollback": two-way reconciliation between local `~/.claude` and GitHub (behind AND ahead/uncommitted), plugin version status, vendored third-party skill drift; `/update full`/`rules`/`rollback`/`restore <name>` apply core changes, `/update vendor <name>`/`marketplace <name>` apply third-party updates (confirm-gated).
- **release** — `/release`, "release", "push this", "commit and push", "get this to github": ONE context-aware publish command for any repo: reads the repo-root SHIP.md playbook and pushes to the right environment (dev/staging/prod). No SHIP.md → stops (lockstep) and helps you build one before anything ships. Never pushes blind against a moved remote — hard-stops if GitHub has commits you don't.
# skill-discovery
When the user says "find skill for X", "what skill handles X", "which skill does X", or "what should I use for X", read `~/.claude/skills/my-skills/TAGLINES.md`, match X against the taglines, and return the single best-matching skill plus its trigger command. This is a routing rule, not a skill — do not invoke the Skill tool.
- **diagnose** — `/diagnose`, "debug this", "can't figure out why", "something's broken", "argue about this bug", "debate the bug": two modes: default = systematic 6-phase diagnosis (reproduce → minimize → hypothesize → instrument → fix → regression-test); `--debate` = 6 Opus personas argue competing root-cause theories → contradiction map → diagnosis + next step.
- **last-30** — `/last-30`: pulls gaining-traction signal (not all-time rankings) from GitHub/HN/YouTube/X filtered to last 30 days only. (routes via SKILL.md description)
- **md-check** — `/md-check`, "check md files", "md hygiene", "fix my md files", "dedupe my docs", "clean up my docs": MD hygiene: line counts, topic-overlap, duplicate-rule detection (read-only audit); `--fix` APPLIES the fixes (dedupe/merge/trim/drift) behind one approve-all gate; `--drift` catches stale descriptions; `--orphans` catches dangling catalog references and unreferenced skills; `--pre <file>` pre-creation gate.
- **prompt-scan** — `/prompt-scan`, "scan my prompts", "refresh learnings": reads all system prompt files + fetches current model release notes → appends dated section to ~/.claude/learnings.md. Feeds /better-prompt.
- **better-prompt** — `/better-prompt`, "sharpen this prompt", "improve my prompt": reads learnings.md, diagnoses a rough prompt, rewrites it with concrete target + scope + success criterion + correct skill route.
- **antislop** — `/antislop`: diagnose AI writing/GUI slop tells against ANTISLOP.md; violations table + verdict, no rewrite. (routes via SKILL.md description)
- **remember-that** — `/remember-that`, "remember that", "save this to memory", "forget that": unified memory manager: add facts, extract from context, delete, move, audit, compile.
- **skill-audit** — `/skill-audit`, "audit my skills", "check my skill library", "find skill gaps", "run skill audit": audits ~/.claude/skills/ across 4 dimensions: bucket fit, component gaps, missing verifiers, trigger conditions. Also builds verifiers and surfaces gotcha gaps. `--research` first pulls live `/last-30` trend radar, then folds gaps into the audit.
- **design** — `/design`, "design this", "new design", "redesign", "fresh look", "start over on the UI", "mock this up", "show me design options", "put a button here", "add a chat box", "drop in a card", "add a component", "drop in a component", "make it match this mockup", "match this mockup", "port this mockup", "make my site look like this mockup": fresh generation only, never preserves existing design: brand seed → Taste generators → 10 Opus mockups (8 paradigms + 2 wild) → slop validator → AI picks best → Chrome auto-opens → you confirm → Opus plan → Sonnet build. `--apply-spec <file>` swaps tokens from a DESIGN.md. Auto-redirects audit/review to `/design-audit`, existing-UI polish to the impeccable plugin. Second mode component-pull: pulls a finished, project-themed component from the project's own library (shadcn/Radix/MUI, auto-detected) or Meta's Astryx design system if none, previews for approval before placing (React-only). Third mode mockup-match: ports an existing surface verbatim to a supplied HTML mockup (new <Name>V2, overlay-screenshot done gate).
- **design-audit** — `/design-audit`, "audit this UI", "review the design", "find design problems": 5 parallel lenses → adversarial verify every Critical → ranked violations table + P0/P1/P2 implementation plan. The audit itself is read-only (zero code, zero mockups) — but saying "fix it" afterward triggers a real 10-mockup fix pipeline scoped to the findings.
- **video-to-kb** — `/video-to-kb`, "ingest video", "process video", "save this talk to my KB": ingests a YouTube/video URL into the Knowledge Base Obsidian vault: saves the raw transcript, then "process it" writes the wiki pages. Distinct from `/watch`, which only transcribes and answers questions without writing to the KB.
- **ingest-docs** — `/ingest-docs`, "ingest docs", "process raw docs", "update context from docs": per-repo doc ingestion pipeline: converts PDF/PPTX/DOCX/images in docs/raw/ → dense .md context files in docs/context/ via markitdown + LLM topic-match (NEW/UPDATE/REPLACE/COMBINE), approval table, manifest tracking.
- **review** — `/review`, "review this", "check the diff", "code health", "run health pass", "review before merge": one mode, always thorough: diff review (correctness/bugs/over-engineering/Karpathy) AND full codebase health pass (fallow + trim + improve), max effort, every time. Bakes in secret scan and antislop on .md changes. One ranked findings list, then it walks each fixable finding past you one at a time. Bare /review in ANY repo also pulls /last-30 trends; in ~/.claude it folds in md-check + skill-audit. "quick review" skips trends; `--auto` skips the walk and batch-applies everything fixable.
- **quick-mockup** — `/quick-mockup`, "quick mockup", "sketch this", "throwaway mockup", "just show me the layout": fast disposable placeholder-only HTML mockup for layout/spatial decisions. ONE file, gray boxes, system-ui font, served over http://, auto-opened in browser. No brand tokens, no slop-judge, no 10-variant pipeline — that's /design's job. Also the concrete tool that satisfies the global "never show ASCII mockups" rule for small/quick asks.
